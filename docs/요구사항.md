### 1. 미션 소개

흉부 X-ray(Chest X-ray, CXR)는 폐렴, 결핵, 폐암 등 다양한 흉부 질환을 진단하는 가장 기본적인 영상 검사다. 현업에서는 영상의학과 전문의가 하루 100~200장의 X-ray를 판독하며, Lunit INSIGHT CXR, VUNO Med-Chest X-ray 등 AI 판독 보조 시스템이 FDA/MFDS 승인을 받아 실제 병원에서 운영되고 있다.
이 미션에서는 NIH ChestX-ray14 데이터셋(112,120장)을 활용하여 14개 흉부 질환을 동시에 탐지하는 Multi-label Classification 시스템을 구축한다. DenseNet-121 기반 예측 모델, Grad-CAM 설명 모듈, Model Ensemble, FastAPI 추론 서버, Streamlit 판독 보조 대시보드까지 End-to-End 의료 영상 AI 시스템을 완성한다.
이 과정에서 Multi-label Classification의 원리, Focal Loss를 통한 Class Imbalance 처리, Operating Point 분석(민감도/특이도 Trade-off), Subgroup Fairness 분석, External Validation을 통한 Domain Shift 검증, Grad-CAM 기반 모델 신뢰성 검증까지 현업 수준의 의료 영상 AI 개발 역량을 확보한다.

본 미션은 3~5인 팀 미션으로 수행한다. 팀원 간 역할 분배를 명확히 하고, README.md에 개인별 기여 요약을 반드시 포함해야 한다.

### 3. 과제 목표

이 과제를 마친 후, 학습자는 아래를 스스로 설명할 수 있어야 한다.

- Multi-label Classification에서 BCEWithLogitsLoss를 사용하는 이유와 Softmax를 사용하면 안 되는 이유를 수식과 함께 설명할 수 있다.
- Focal Loss의 감마 파라미터가 Easy/Hard Example의 Loss 기여도를 어떻게 조절하는지 설명할 수 있다.
- Patient-wise Split의 필요성과 Random Split 시 발생하는 데이터 누수 메커니즘을 설명할 수 있다.
- 스크리닝 vs 확진 용도에 따른 Operating Point 선택 전략을 민감도/특이도 관점에서 설명할 수 있다.
- External Validation에서 Domain Shift가 발생하는 원인과 대응 전략을 설명할 수 있다.
- Grad-CAM 히트맵이 폐 영역을 벗어나는 경우 Shortcut Learning 가능성을 판단하고 해석할 수 있다.

### 4. 기능 요구 사항

다음 요구사항을 모두 만족해야 한다.

1. **데이터 전처리**
    - NIH ChestX-ray14 데이터셋(112,120장)을 로드해야 한다.
    - pydicom을 활용해 DICOM 샘플 파일의 메타데이터와 이미지를 파싱해야 한다.
        - DICOM 샘플 사용 시 개인식별정보(PHI) 제거 및 비공개 저장 금지 원칙을 준수해야 한다.
    - 14개 질환에 대해 Multi-hot Encoding을 수행하고, 'No Finding'을 별도로 처리해야 한다.
    - 이미지를 모델 입력 크기(224x224)로 리사이즈하고 ImageNet 평균/표준편차로 정규화해야 한다.
    - CLAHE를 적용하고, 적용 전후 폐 병변(Nodule 등)의 시인성 변화를 비교 분석하여 문서화해야 한다.
    - 이미지 품질 검증: 너무 밝거나 어두운 이미지를 필터링하고, 필터링된 이미지 수를 기록해야 한다.
    - 데이터 분할 절차 (표준안)
        - 전체 데이터를 Patient ID 기준으로 Train/Test (85:15)로 분리하여 Test를 고정한다.
        - Train 세트 내에서 5-Fold GroupKFold (Patient-wise)로 튜닝/검증을 수행한다.
        - 최종 성능은 고정된 Test 세트에서 1회 평가한다.
    - 동일 환자의 이미지가 서로 다른 세트에 포함되지 않음을 검증하고 문서화해야 한다.
    - 질환별 유병률(Prevalence)을 계산하고 표로 정리해야 한다.
    - Co-occurrence Matrix(동시 발생 빈도)를 히트맵으로 시각화해야 한다.
2. **모델 학습**
    - DenseNet-121 (ImageNet Pretrained)을 베이스라인으로 사용해야 한다.
    - 마지막 레이어를 14개 출력(Sigmoid)으로 변경해야 한다.
    - Focal Loss를 직접 구현해야 한다 (외부 라이브러리 사용 금지).
    - 감마(gamma) 파라미터 0, 1, 2에 따른 성능 차이를 실험하고 표로 정리해야 한다.
        - 예시
            | **gamma** | **Mean AUROC** | **Mean AUPRC** | **ECE** | **해석 한 줄** |
            | --- | --- | --- | --- | --- |
            | 0 | (BCE 동일) |  |  |  |
            | 1 |  |  |  |  |
            | 2 |  |  |  |  |
    - 최적 gamma 선택 근거를 문서화해야 한다.
    - 질환별 유병률 기반으로 pos_weight를 계산하고 적용해야 한다.
    - AdamW Optimizer 및 Cosine Annealing Scheduler를 사용해야 한다.
    - 5-Fold GroupKFold (Patient-wise) Cross Validation을 수행해야 한다.
    - Early Stopping (patience=5, monitor=val_auroc)을 적용해야 한다.
    - Fold별 성능을 기록하고 Mean ± Std를 계산해야 한다.
3. **Test-Time Augmentation 및 Ensemble**
    - Test-Time Augmentation: 추론 시 Horizontal Flip을 적용해야 한다.
        - TTA 주의사항: TTA는 성능 개선 여부 검증 목적으로 적용한다. 좌우 위치 의존 질환(Cardiomegaly, 좌측 폐열 등)의 Grad-CAM 해석 시 H-Flip이 위치 정보를 변경할 수 있음을 인지하고, 해당 제한사항을 문서화해야 한다.
    - TTA 적용 전후 Mean AUROC 변화를 비교하고 문서화해야 한다.
    - DenseNet-121 외에 EfficientNet-B0 또는 B4를 추가 학습해야 한다.
    - Soft Voting 방식의 2-Model Ensemble을 구현해야 한다.
    - Single Model 대비 Ensemble 성능 향상 폭을 문서화해야 한다.
4. **성능 평가 및 검증**
    - 14개 질환별 AUROC를 계산하고 표로 정리해야 한다.
    - 14개 질환별 AUPRC를 계산하고 표로 정리해야 한다.
    - Mean AUROC 0.80 이상을 달성해야 한다.
    - Youden's J 통계량 기준으로 질환별 최적 Cut-off Threshold를 산출해야 한다.
    - 민감도 90% 고정 시 특이도(Spec@Sens90)를 계산해야 한다.
    - 특이도 90% 고정 시 민감도(Sens@Spec90)를 계산해야 한다.
    - 스크리닝 용도 vs 확진 용도에 따른 Operating Point 선택 근거를 문서화해야 한다.
    - Calibration Curve (Reliability Diagram)를 생성해야 한다.
    - ECE (Expected Calibration Error)를 계산하고 0.10 이하를 목표로 해야 한다.
    - ECE가 0.10 초과 시 Temperature Scaling을 적용하고 보정 전후를 비교해야 한다.
    - 성별(Male/Female)별 Mean AUROC를 비교해야 한다.
    - 연령대(0-40, 40-60, 60+)별 Mean AUROC를 비교해야 한다.
    - View Position(PA/AP)별 Mean AUROC를 비교해야 한다.
    - Subgroup 간 성능 차이가 10% 이상인 경우 원인을 분석하여 문서화해야 한다.
    - 학습에 사용하지 않은 외부 데이터셋(CheXpert 또는 PadChest)에서 성능을 테스트해야 한다.
    - Internal vs External 성능 하락 폭을 계산해야 한다.
    - Domain Shift 원인을 3가지 이상 분석하여 문서화해야 한다.
5. **설명 가능 AI (XAI) 및 에러 분석**
    - 마지막 Convolution Layer를 타겟으로 Grad-CAM을 구현해야 한다.
    - 양성 예측(확률 > Threshold) 질환에 대해 히트맵을 생성해야 한다.
    - 원본 이미지 위에 히트맵을 오버레이하여 시각화해야 한다.
    - 히트맵이 폐 영역을 벗어나는 케이스를 5건 이상 샘플링해야 한다.
    - Shortcut Learning 여부(뼈, 의료기기, 텍스트 등 오인식)를 분석해야 한다.
    - False Positive 케이스를 5건 이상 선정하고 Grad-CAM으로 원인을 분석해야 한다.
    - False Negative 케이스를 5건 이상 선정하고 Grad-CAM으로 원인을 분석해야 한다.
    - 에러 케이스의 공통 패턴과 개선 방향을 문서화해야 한다.
6. **모델 서빙 API**
    - FastAPI를 사용하여 API 서버를 구현해야 한다.
    - GET /health 엔드포인트: 서버 상태와 모델 로드 여부를 반환해야 한다.
    - POST /predict 엔드포인트를 구현해야 한다.
    - 입력: X-ray 이미지 파일 (Binary, PNG/JPEG)
    - 출력 (JSON): 14개 질환별 예측 확률, 탐지된 주요 질환명, Top-1 질환의 Grad-CAM 히트맵 (Base64)
    - Pydantic을 활용해 입출력 스키마를 정의해야 한다.
    - 추론 시간은 이미지 1장당 500ms 이내여야 한다.
    - Swagger UI(/docs)가 자동 생성되어야 한다.
    - 잘못된 이미지 포맷에 대한 에러 메시지를 반환해야 한다.
7. **판독 보조 대시보드**
    - Streamlit을 사용하여 대시보드를 구현해야 한다.
    - 이미지 업로드: PNG, JPEG 파일을 업로드할 수 있어야 한다.
    - 결과 시각화: 원본 이미지 + Grad-CAM 오버레이를 표시해야 한다.
    - 14개 질환의 확률을 수평 막대 그래프로 표시해야 한다.
    - 위험도에 따라 색상을 구분해야 한다 (빨강: 0.5 이상, 노랑: 0.3~0.5, 초록: 0.3 미만).
    - 탐지된 질환 목록을 하이라이트해야 한다.
    - 대시보드는 모델(.pth)을 직접 로드하지 않고, 반드시 API 서버와 HTTP 통신해야 한다.

### 5. 보너스 과제 (선택)

1. Uncertainty Estimation (불확실성 추정)
    - 해야 할 작업 : Monte Carlo Dropout 기법을 적용하여 모델이 자신의 예측을 얼마나 확신하는지(Epistemic Uncertainty)를 수치화한다. 불확실성이 높은 케이스를 "판독 주의 필요"로 표시한다.
    - 배움 포인트 : 의료 AI에서 불확실성 정량화의 중요성과 임상 의사결정 지원 방식을 경험한다.
2. Vision Transformer 적용
    - 해야 할 작업 : DenseNet/EfficientNet 외에 ViT(Vision Transformer)를 추가로 학습하고 CNN 기반 모델과 성능을 비교한다. Attention Map을 시각화하고 Grad-CAM과 비교 분석한다.
    - 배움 포인트 : Transformer 아키텍처의 의료 영상 적용 가능성과 Self-Attention 메커니즘을 경험한다.
3. Out-of-Distribution (OOD) Detection
    - 해야 할 작업 : 흉부 X-ray가 아닌 다른 이미지(예: 손 뼈 사진, 일반 사진)가 들어왔을 때 "분석 불가"로 걸러내는 로직을 추가한다. Softmax Confidence 또는 Mahalanobis Distance 기반으로 구현한다.
    - 배움 포인트 : 배포 환경에서의 모델 안전성 확보와 입력 검증의 중요성을 경험한다.

### 7. 제약 사항

데이터

- NIH ChestX-ray14 데이터셋을 메인으로 사용해야 한다.
    - NIH ChestX-ray14 데이터셋의 라벨은 NLP 기반 자동 추출으로 생성되어 노이즈가 포함되어 있다. 학습자는 다음을 인지해야 한다.
        - Ground Truth 불확실성: 자동 라벨링 특성상 False Label이 존재할 수 있으며, 이는 모델 성능의 상한선에 영향을 미친다.
        - 에러 분석 시 해석: FP/FN 분석 시 라벨 오류 가능성을 함께 고려하여 “모델 오류” vs “라벨 오류”를 구분하여 문서화해야 한다.
        - 권장 해석 방식: 성능 지표를 보고할 때 “NIH 라벨 기준”임을 명시하고, 임상적 성능과 구분해야 한다.
- External Validation용 데이터셋 (CheXpert 또는 PadChest)을 1개 이상 사용해야 한다.
- 테스트 데이터의 라벨을 학습에 사용하는 것을 금지한다.
- Patient ID 기준 분리로 데이터 누수를 방지해야 한다.

아키텍처

- 프론트엔드(Streamlit) 코드에서 모델 파일을 직접 로드하는 것을 금지한다.
- 반드시 API 통신을 통해서만 추론 결과를 받아야 한다 (MSA 구조 강제).

코드 품질

- GitHub의 Issue와 Pull Request 기능을 적극 활용하여 개발 이력을 남겨야 한다.
- 설정값은 환경 변수 또는 config 파일로 분리해야 한다.
- Focal Loss는 직접 구현해야 한다 (외부 라이브러리 사용 금지).

성능 기준

- Mean AUROC 0.80 이상이어야 한다.
- 추론 시간은 이미지 1장당 0.5초 이내여야 한다.
- ECE ≤ 0.10 (Temperature Scaling 적용 후)

제출 형태

- 소스 코드가 포함된 GitHub Repository URL을 제출한다.
- Private Repo일 경우 채점자 계정 초대 필수.

의료 AI 특이사항

- 본 시스템은 교육 목적으로 개발되며 실제 임상 진단에 사용할 수 없다.
- AI 예측 결과는 참고용이며 최종 진단은 의료 전문가가 수행해야 한다.

- [ ]  추론 시간 : 이미지 1장당 추론 시간이 500ms 이내인가?
- [ ]  대시보드 연동 : Streamlit 대시보드에서 API 서버와 통신하여 결과가 표시되는가?
- [ ]  위험도 색상 : 확률에 따른 색상 구분(빨강/노랑/초록)이 올바르게 동작하는가?

코드 구조 및 설계

- [ ]  역할 분리 : 데이터 전처리, 모델 학습, 추론(API), UI 코드가 모듈별로 분리되어 있는가?
- [ ]  누수 방지 : Train/Val/Test 분리 시 Patient ID 기준 GroupKFold를 사용했고, 누수 없음이 검증되었는가?
- [ ]  설정 관리 : 모델 경로, Hyperparameter 등이 config 파일 또는 환경 변수로 관리되는가?
- [ ]  테스트 존재 : pytest 유닛 테스트가 3개 이상 존재하고 통과하는가?
- [ ]  MSA 준수 : Streamlit 코드에서 모델 파일을 직접 로드하지 않고, API 통신만 사용하는가?

핵심 기술 원리 적용

- [ ]  CLAHE 분석 : CLAHE 적용 전후 병변 시인성 비교 분석이 문서화되어 있는가?
- [ ]  Focal Loss 실험 : gamma 파라미터(0, 1, 2)에 따른 성능 차이 실험 결과와 선택 근거가 있는가?
- [ ]  Operating Point 분석 : Youden's J, Sensitivity 90%, Specificity 90% 기준의 임계값과 스크리닝/확진 용도별 선택 근거가 문서화되어 있는가?
- [ ]  Calibration 검증 : ECE가 계산되고 0.10 이하인가? 초과 시 Temperature Scaling이 적용되었는가?
- [ ]  Subgroup Analysis : 성별/연령대/View Position별 성능 비교와 10% 이상 차이 시 원인 분석이 있는가?
- [ ]  External Validation : 외부 데이터셋 테스트 결과와 Domain Shift 원인 3가지 이상 분석이 있는가?
- [ ]  에러 케이스 분석 : FP/FN 각 5건 이상에 대한 Grad-CAM 분석과 Shortcut Learning 여부 검토가 있는가?
